FAQ — eng ko‘p beriladigan savollar

FraudShield AI tizimi bo‘yicha tez-tez so‘raladigan savollar va qisqa javoblar.

Taqdimot vaqtida hakamlar va bank mutaxassislari tezda tushunishi uchun mo‘ljallangan.

Savollar va javoblar

Tizim bank kartalari va hisob raqamlari bo‘yicha firibgarlik tranzaksiyalarini real vaqt rejimida aniqlashga yordam beradi. Model katta miqdordagi yoki noodatiy operatsiyalarni flag qilib, mijoz mablag‘ini himoya qilishni avtomatlashtiradi.
Backend: Python (Flask), Machine Learning: scikit-learn (Random Forest), ma’lumotlar tahlili: pandas, numpy. Model REST API ko‘rinishida taqdim etiladi va istalgan bank tizimiga HTTP orqali ulanishi mumkin.
Ha, model strukturasi real tranzaksiya loglari bilan ishlashga moslab tuzilgan. Hozir prototip bosqichida ochiq datasetlar asosida o‘qitilgan, lekin bankning tarixiy ma’lumotlari bilan qayta o‘qitish orqali aniq natijaga erishish mumkin.
Model natijasini qattiq bloklashdan ko‘ra “risk balli” sifatida ishlatamiz. High-risk operatsiyalar qo‘shimcha verifikatsiyaga yuboriladi (SMS, yuzni tasdiqlash, qo‘ng‘iroq). Threshold qiymatini bank siyosatiga qarab moslashtirish mumkin.
Random Forest modeli optimallashtirilgan holda millisekundlar ichida javob qaytaradi. Prototip darajasida ham real vaqt rejimida API ga so‘rov yuborish va javob olish 50–100 ms oralig‘ida bo‘lishi kutilmoqda (infratuzilmaga bog‘liq).
Asosiy AI komponenti – bu firibgarlik ehtimolini baholovchi Random Forest modeli. Keyingi bosqichlarda anomaliya deteksiyasi, sequence modelling (LSTM/Transformer) va graf asosidagi fraud tarmoqlarini tahlil qilish kabi modullarni ham qo‘shish rejalashtirilgan.
Integratsiya RESTful API orqali amalga oshiriladi. Tranzaksiya so‘rovi paydo bo‘lganda bank tizimi bizning API’ga minimal feature’lar to‘plami bilan so‘rov yuboradi va 0–1 oralig‘ida risk ballini oladi. Javobga qarab operatsiyani tasdiqlash, kutishga qo‘yish yoki bloklash mumkin.
Model faqat texnik atributlar bilan ishlaydi (summa, vaqt, balans farqlari va h.k.). Shaxsiy ma’lumotlarni (F.I.O, PINFL, telefon) uzatmaslik mumkin. API HTTPS orqali himoyalangan kanal ustida ishlashi ko‘zda tutilgan.
Jamoamiz data science va cybersecurity yo‘nalishida tajribaga ega. Pilot bosqichda modelni biz o‘qitamiz va monitoring qilamiz. Keyinchalik bankning ichki jamoasiga hujjatlar va pipeline berilib, ularning o‘zlari modelni muntazam yangilab borishlari mumkin.
Asosiy fokus – bank kartalari va elektron to‘lovlar. Lekin shu yondashuvni onlayn savdo platformalari, fintex servislar yoki mikromoliya tashkilotlaridagi tranzaksiya xavfini baholashda ham moslashtirish mumkin.
